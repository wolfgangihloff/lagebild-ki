menu_book Glossar

  • IDE – Integrated Development Environment (z.B. VS Code)
  • Agent – KI die eigenständig Aktionen ausführt
  • Prompt – Anweisung/Eingabe an die KI
  • Context Window – Wie viel Text die KI gleichzeitig „sehen“ kann
  • Commit – Gesicherter Zwischenstand im Code
  • Monorepo – Ein Repository für Frontend + Backend
  • Walking Skeleton – Minimale End-to-End-Version
  • TDD – Test-Driven Development
  • CI/CD – Automatisches Bauen & Ausliefern
  • Container – Isolierte Laufzeitumgebung (Docker)

payments Was kostet das?

  • money_off Free Tier – Codex, Kiro, Cursor Free, Claude Free – begrenzt aber zum Ausprobieren genug
  • credit_card Pro/Max – ~20–200 $/Monat pro Tool
  • api API / Pay-per-Use – Flexibel, aber Vorsicht: Agents können unkontrolliert Kosten erzeugen

Tipp: Mit Flat Rate starten, bis man seinen Verbrauch kennt.

Wolfgang nutzt Claude Pro Max (~200$/mo) + Codex (~200$/mo). Erfahrung: Freitags ist Claude-Quota oft aufgebraucht, dann Wechsel auf Codex.

security Sicherheit & Zugriff

  • folder Was sieht der Agent? – Lokale Dateien im Projektordner. Zugriff konfigurierbar
  • cloud Wo gehen Daten hin? – API-Provider. Keine dauerhafte Speicherung bei Pro-Accounts
  • lock Best Practices:
    • Verzeichnis-Zugriff einschränken (ein Projekt pro Session)
    • .env nie committen – .gitignore pflegen
    • Secrets Manager nutzen (1Password CLI, Vault)
    • Default: alles aus, gezielt freigeben
Tipp aus Session 1: Ein Teilnehmer schaltet standardmaessig alle Dateizugriffe aus. "Wenn man API-Keys auf GitHub pusht, sind sie in unter 5 Minuten kompromittiert."

science Prototyp vs. Production

Prototyp 2–4 Stunden · zeigt Machbarkeit · hat Bugs · beweist den Ansatz Production Tests · Security Review · Monitoring · Logging · Doku · Edge Cases
  • KI beschleunigt 10x – ersetzt aber nicht den Qualitätsprozess
  • Was ihr in der Session seht ist ein funktionierender Beweis, kein fertiges Produkt
  • Der Weg zur Production braucht die gleiche Sorgfalt – nur schneller
Wichtig: Diese Erwartung frueh setzen, nicht erst am Ende wenn Bugs auftauchen.
