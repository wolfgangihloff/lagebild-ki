Wie wir das bauen

20 min Requirements – KI schreibt die Spec aus unserer Feature-Liste 25 min Anwendung – KI baut das Projekt (API-Anbindung, Logik, Tests)   Pause 20 min Dokumentation – KI generiert README, Setup Guide, Architektur-Doc 20 min Infrastruktur – CI/CD Pipeline, Container, Deployment auf k3s 15 min Agent als PM – Agent bekommt nächstes Feature, macht alles allein

Ein Projekt. Fünf Perspektiven. Alles live in VS Code.

save Zwischenstände sichern

  • Agent-Fehler ziehen sich durch das gesamte System – DB, API, UI gleichzeitig betroffen
  • Nach jedem stabilen Schritt committen – das ist euer Sicherheitsnetz
  • Fortschritt = stabile Zwischenstände, nicht Menge an generiertem Code

Grundregel: Nach jedem validierten Schritt ein Commit – egal welches Tool.

restart_alt Sessions bewusst neu starten

  • Kontextfenster ist begrenzt – Qualität sinkt mit der Sessionlänge
  • Symptome: widersprüchliche Antworten, Wiederholungen, Scheinfortschritt
  • Kurze Übergabe mit Ziel, Regeln und offenem Task – max. 10–20 Zeilen

Neustart ist kein Rückschritt, sondern Re-Fokussierung.

shield Leitplanken definieren

  • Agents haben kein Langzeitgedächtnis über Sessions hinweg
  • Zentrale Regeldatei anlegen: agents.md, .cursorrules, CLAUDE.md
  • Schlank halten – zu viel Kontext reduziert die Effizienz

Wie ein Mitarbeiterhandbuch – Standards definieren, kontinuierlich pflegen.

view_agenda Kleine Arbeitspakete schneiden

  • Der Fehler-Radius wächst mit der Aufgabengröße
  • Jeder Schritt: umsetzen → testen → committen
  • Gute Task-Definition ersetzt technische Detailarbeit

Scope begrenzen: „Implementiere nur Schritt 1. Keine UI, keine API.“

verified Qualität explizit einfordern

  • Agents liefern nur den Happy Path – ohne explizite Anweisung
  • Fehlerhandling, Sicherheit, Datenschutz aktiv beauftragen
  • Demo sieht gut aus – Probleme kommen erst im Betrieb

Explizit: „Ergänze Fehlerbehandlung. Nutzer sehen nur eigene Daten. Keine sensiblen Logs.“

supervisor_account Neue Rolle: AI Supervisor

  • Rolle verschiebt sich: Koordination → Steuerung von AI-Prozessen
  • Erst planen, dann umsetzen, dann reviewen – je Schritt
  • Steuerung vor dem Fehler statt danach

„Erstelle zuerst einen Plan. Ich gebe frei. Stoppe nach jedem Schritt für Review.“
